# Sales Dashboard v2

Полный дашборд продаж с умной аналитикой: автоинсайты, прогноз спроса,
cross-sell, сравнение периодов (MoM/YoY/WoW), рейтинг точек, сезонность категорий.

## Структура

```
dashboard_v2/
├── app2.py                      ← основной Streamlit-дашборд
├── category_mapping.py          ← 8 групп + нормализация подкатегорий
├── build_refactored_excel.py    ← разовый скрипт: обновляет docs/Итоговый_отчет1.xlsx из оригинала
├── requirements.txt
├── runtime.txt
├── .gitignore
└── docs/                        ← отдаётся через GitHub Pages
    ├── Итоговый_отчет1.xlsx     ← рефакторенная база (с колонкой Группа)
    └── разбивка_наборов.xlsx
```

```bash

```

## GitHub Pages

Чтобы Excel подтягивался с Pages:

1. Settings → Pages → Source: **Deploy from a branch** → **main / `/docs`** → Save.
2. После первого деплоя URL будет: `https://monkeydjack.github.io/yahya-sales-dashboard/`
3. Файлы доступны напрямую:
   - `.../Итоговый_отчет1.xlsx`
   - `.../разбивка_наборов.xlsx`

Приложение уже настроено на этот URL в константе `GH_PAGES_EXCEL`.

## Обновление данных из 1С

1. Открой оригинал `Итоговый_отчет1.xlsx` в корне репо (не в `docs/`).
2. Вставь новые строки в лист «база» в конец таблицы.
   - Колонки B–M заполни из 1С.
   - Колонки A, L, M, N должны подтянуться VLOOKUP'ом — если Excel не протянул,
     выдели последнюю строку с формулами и перетащи вниз.
3. Если появились новые товары — добавь их в лист «Лист1»:
   Номенклатура │ Точки │ Категория │ Подкатегория │ Группа.
4. Запусти пересборку docs/-копии:
   ```bash
   python dashboard_v2/build_refactored_excel.py
   ```
   Это нормализует подкатегории, досчитает группу и сохранит файл в `dashboard_v2/docs/`.
5. `git add dashboard_v2/docs/Итоговый_отчет1.xlsx && git commit -m "data: refresh" && git push`.
6. Через 1–2 минуты GitHub Pages отдаст новую версию, дашборд её подтянет.
