Файлы раздаются через **GitHub Pages** репозитория `MonkeyDjack/yahya-sales-dashboard`

> ежемесячным аппендом, структура — стабильная.

## Как получить

```python
import io, requests, pandas as pd

URL = "https://monkeydjack.github.io/yahya-sales-dashboard/база.parquet"  # requests сам закодирует кириллицу

r = requests.get(URL, timeout=120)
r.raise_for_status()
df = pd.read_parquet(io.BytesIO(r.content), engine="pyarrow")
```

**Обязательная очистка после загрузки** (`Дата` хранится СТРОКОЙ `dd.mm.yyyy`!):

```python
df["Дата"] = pd.to_datetime(df["Дата"], errors="coerce", dayfirst=True).dt.normalize()
for col in ["Количество", "Сумма", "Цена"]:
    df[col] = pd.to_numeric(df[col], errors="coerce")
```

В дашборде это делает `dashboard_v2/core/data.py: load_sales()` (плюс
`category_mapping.apply_mapping` — досчёт Группы). Кеш — `@st.cache_data(ttl=3600)`,
поэтому обновления parquet проявляются в дашборде с задержкой до 1 часа
(или по кнопке «🔄 Обновить данные» в sidebar).

PowerShell-скачивание: `Invoke-WebRequest -Uri $url -OutFile база.parquet`.

## Схема `база.parquet`

Одна строка = одна позиция в чеке.

| Колонка | Тип в файле | Описание / грабли |
|---|---|---|
| `Дата` | **str** `'22.01.2026'` | дата продажи; парсить с `dayfirst=True` |
| `Время` | str `'16:38:37'` | время чека, формат `%H:%M:%S` |
| `Филиал` | str | 7 филиалов, см. ниже |
| `Точки` | str | цех: `Магазин` / `Кухня` / `Бар` |
| `Номенклатура` | str | SKU, 687 уникальных |
| `Склад/Товар` | str `'Чек 000217923'` | **идентификатор чека**. Чеки = `nunique()` после трима пробелов; ~175.6 тыс уникальных |
| `Количество` | float | штуки (для весовых — может быть дробным) |
| `Цена` | float | цена позиции |
| `Сумма` | float | выручка позиции = Цена × Количество |
| `Месяц` | str `'Январь'` | русское название — для группировок лучше брать из `Дата` |
| `Год` | int | 2025 / 2026 |
| `Группа` | str | 8 верхнеуровневых групп (канон — `dashboard_v2/category_mapping.py`) |
| `Категория` | str | ~61 уникальная |
| `Подкатегория` | str | ~108 уникальных |

## Филиалы и события (критично для сравнений!)

| Филиал | Работает с | Выручка (вся история) | Примечание |
|---|---|---|---|
| ЦУМ | 01.01.2025 | 51.1 млн | крупнейший |
| ЭРКИНДИК | 02.01.2025 | 47.2 млн | |
| АЗИЯ МОЛЛ | 01.01.2025 | 28.4 млн | **реконструкция 08.02–01.03.2026 (22 дня)** |
| АЭРОПОРТ | 01.01.2025 | 25.2 млн | |
| Бишкек Парк | 01.01.2025 | 9.1 млн | |
| NGROUP | 02.01.2026 | 3.0 млн | новый |
| АЗБУКА | 01.01.2026 | 2.1 млн | новый |

Правила честных сравнений (реализованы в `dashboard_v2/core/events.py` и `core/periods.py`):
- **Год-к-году** NGROUP и АЗБУКА сравнивать нельзя — их не было в 2025.
- Февраль 2026 у АЗИЯ МОЛЛ несопоставим ни с чем (22 дня простоя).
- LFL-логика: филиал участвует в сравнении, только если работал ≥95% дней
  **в обоих** периодах (`BRANCH_EVENTS` + `lfl_split`). **Новые открытия/закрытия
  филиалов — добавлять в `BRANCH_EVENTS`!**
- Δ% считать по среднему/день (периоды бывают неравной длины).
- YoY = сдвиг **−364 дня** (52 недели — сохраняет день недели), не −365.
- Сезонность: июнь–декабрь представлены только 2025 годом, январь–май — двумя.

## Справочники

### `себестоимость.parquet` (352 SKU)

| Колонка | Описание |
|---|---|
| `Номенклатура` | SKU как в базе продаж |
| `Себес сырья` | сырьевая себестоимость |
| `ПНР` | прочие накладные расходы |
| `Себес полн.` | **полная себестоимость — её использовать для маржи** |
| `Розница` | розничная цена из прайса |
| `Сегмент` | `Кухня и бар` / шоколад и т.п. |
| `Источник` | из какого xlsx-прайса взято |
| `Номенклатура_прайс` | оригинальное имя в прайсе (до нормализации) |

Маржа в дашборде: `(Сумма/Количество − Себес полн.) × Количество` по SKU.
**Покрытие себесом ~84.5% выручки.** Без себеса: сезонные SKU (нет файла),
шококарандаши (нет в прайсе), снятые с производства.
Пересборка: `python dashboard_v2/build_cost_reference.py` (из xlsx-прайсов в `D:/claude/`).

### `list1.parquet` (688 SKU)
Колонки: `Номенклатура`, `Точки`, `Категория`, `Подкатегория`, `Группа` —
канонический справочник иерархии SKU.

### `разбивка_наборов.xlsx` (BOM)
Колонки: `Набор`, `Компонент`, `Кол`. `Кол < 1` трактуется как килограммы,
иначе штуки. Сезонные наборы (23 февраля, 8 марта, новый год, рамадан…)
при загрузке отфильтровываются по ключевым словам (`core/config.py: SEASONAL_KEYWORDS`).

## Известные особенности данных

1. **SKU с суффиксом «В»** в конце имени = «Вынос». Себес и розница те же,
   что у базовой версии без «В» (учтено в `build_cost_reference.py`).
2. **Категории чинились скриптом** `dashboard_v2/fix_categories.py`: багованный
   импорт из 1С ставил части строк fallback «Конфеты / Корпусные конфеты»;
   канон восстановлен по моде SKU. «Без подкатегории» → имя категории.
   У «Нац.коллекция (соленая карамель и талкан)» — ручной оверрайд.
   **После каждого нового импорта из 1С скрипт надо прогонять заново.**
3. **Шоколадные наборы из группы «Наборы и коробки»** (Нац.коллекция, Корпусная/
   Нарезная/Коктельная коллекция, дубайские мини плитки, Слива-Грецкий НК, Ынтымак)
   бизнес-логически — шоколад; в шоколадных отчётах включаются явным списком префиксов.
4. Один SKU всегда принадлежит одной точке (`Магазин`/`Кухня`/`Бар`) — связка
   берётся по первому вхождению.

## Как данные обновляются

- **Ежемесячно**: выгрузка из 1С → скрипт `dashboard_v2/append_*.py` дописывает
  месяц в `docs/база.parquet` → `fix_categories.py` → commit + push в `main` →
  GitHub Pages раздаёт новую версию → дашборд подхватывает в течение часа.
- Перед массовыми правками создаются бэкапы `docs/*.bak_*` — в репо **не коммитятся**.
- Прайсы себеса обновляются реже: новый xlsx в `D:/claude/` → `build_cost_reference.py`.
